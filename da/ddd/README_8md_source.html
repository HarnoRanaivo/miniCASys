1 miniCASys
2 =========
3 
4 Minicas, projet d'ALinéA (Algèbre Linéaire), L2S4, Université de Strasbourg. Mini interpréteur de commandes correspondant à des opérations classiques d'algèbre linéaire.
5 
6 Télécharger et compiler « Chasseur de Prim »
7 --------------------------------------------
8 ### Pré-requis ###
9 Tout d'abord, il vous faudra les outils standards de développement GNU, ainsi que git. Ce programme tire également parti de ```gnuplot```.
10 
11 Pour Ubuntu :
12 ````bash
13 $ apt-get install build-essentials git-core gnuplot
14 ````
15 
16 Pour Fedora :
17 ````bash
18 $ yum groupinstall "Development Tools"
19 $ yum install git-all gnuplot
20 ````
21 
22 ### Récupérer le projet ###
23 Pour récupérer le projet dans le dossier de votre choix :
24 ````bash
25 $ git clone git://github.com/remove/miniCASys.git [dossier]
26 ````
27 Ensuite, pour récupérer les changements :
28 ````bash
29 $ git pull
30 ````
31 
32 ### Compiler ###
33 Un makefile est fourni, il vous suffit de vous placer à la racine du projet et de lancer make :
34 ````bash
35 $ make
36 ````
37 
38 Lancer Minicas
39 -----------------------
40 ### Mode interactif ###
41 Une fois le programme compilé, il suffit de lancer le binaire disponible dans ```bin``` :
42 ````bash
43 $ ledit bin/minicas
44 ````
45 ```ledit``` est optionnel, ce programme permet d'obtenir un prompt plus agréable (historique, complétion des noms de fichiers, ...). Pour obtenir de l'aide, entrez la commande ```aide```. Pour quitter le programme, entrez la commande ```quitter``` ou ```q```, puis confirmez votre décision par ```oui```.
46 
47 ### Mode non interactif ###
48 Il est possible de lancer des scripts :
49 ````bash
50 $ bin/minicas <fichier>
51 ````
52 Ceux-ci doivent contenir une commande par ligne. Les commandes valides sont toutes celles qui sont disponibles dans le mode interactif.
53 
54 ### Version ###
55 Pour connaître la version de ```minicas``` que vous utilisez :
56 ````bash
57 $ bin/minicas --version
58 ````
59 
60 Branches
61 --------
62 Les versions stables se trouvent dans la branche ````master````, la branche ````dev```` contient la version de développement du projet. Pour passer d'une branche à l'autre :
63 ````bash
64 $ git checkout <branche>
65 ````
